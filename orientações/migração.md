Pessoal, passando um alinhamento importante sobre o fluxo de casos relacionados a possíveis erros de migração :warning:

Como era antes: Até então, quando o cliente sinalizava algum problema de migração, a orientação era direcionar o caso para CS. Porém, após discussões internas entre as áreas, alinhamos que, por enquanto, esse tipo de demanda deve seguir um novo fluxo.

:atencao1:Como será a partir de agora: sempre que o cliente reportar um possível erro de migração, o caso deve ser direcionado aos gatekeepers (@Ciprianoou @Ithalo Souza).
Os gatekeepers serão responsáveis por realizar uma análise mais aprofundada do ocorrido para validar se a situação é, de fato, um erro de migração.

Se não for identificado como erro de migração: o caso deve seguir o fluxo de tratativa padrão.
Se for confirmado erro de migração: os gatekeepers devem escalar o caso para o N2.


Ao receber o caso, o N2 fará a validação final e a classificação como elegível, seguindo com as próximas etapas necessárias junto ao time de migração.
